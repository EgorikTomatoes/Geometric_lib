# План тестирования

## 1. Цели и задачи тестирования
- Подтвердить корректность вычисления площади и периметра круга и квадрата.
- Зафиксировать ожидаемое поведение функций при типичных значениях и нуле.
- Обеспечить возможность автоматического регрессионного прогона через `unittest`.

## 2. Описание тестируемого продукта
Тестируются функции модулей `circle.py` и `square.py`, рассчитывающие:
- `area(r)` и `perimeter(r)` для круга по радиусу;
- `area(a)` и `perimeter(a)` для квадрата по длине стороны.

## 3. Область тестирования
- Нормальные значения аргументов (целые и дробные числа).
- Граничное значение 0 для проверки корректной обработки нулевых длин.
- Расчёт с использованием числа π для окружности.

## 4. Стратегия тестирования
- Тип тестирования: модульное (`unittest`).
- Подход: проверка точности вычислений по известным формулам (ожидаемые значения вычисляются в тестах).
- Метрики: количество пройденных/упавших тестов и их процент.
- Среда: Python 3.x без внешних зависимостей.

## 5. Критерии приемки
- Все тесты проходят успешно (`OK` в выводе `python -m unittest`).
- Отклонение от ожидаемых значений не превышает допуска `assertAlmostEqual` для операций с π.
- Отсутствуют необработанные исключения при запуске тестов.

## 6. Ожидаемые результаты
- Автоматизированный набор тестов в `tests/test_geometry.py`.
- Итоговый отчёт запуска (локально или в CI) со статусом `OK` при корректной работе.
- Возможность повторного регрессионного прогона при изменениях кода без ручных расчётов.
